# 前端验证清单

## 默认门禁
- 最少执行：
  - `npm run lint`
  - `npm run typecheck`
- 涉及构建、资源、Capacitor、原生插件、样式出包时，再执行：
  - `npm run build`
- 涉及容器能力时，再执行：
  - `npm run cap:sync`
  - 必要时 `npm run cap:sync:ios` 或 `npm run cap:sync:android`

## 交付目标提醒
- 前端最终目标是 iOS / Android 移动应用，不是只在浏览器可用。
- 浏览器验证只能证明一部分，不等于原生容器交付结论。

## 场景与路由检查
- `Hero -> Enter -> /dream/new` 主流程是否仍成立
- `/gallery`、`/my-dreams`、`/archive` 进入后是否还能保持一级页语义
- 顶部“首页”类动作是否错误回到 `/`
- 一级底部导航是否只出现在一级页

## 首页与导航检查
- 首页 safe area 顶部是否正确留白
- 底部导航是否避开底部 safe area
- 激活滑块是否平滑滑动
- 只能拖当前激活 tab，整条导航不能被拖走
- 跨两格切换是否正常
- 竖向滚动时是否误抢手势

## 运行时检查
- 键盘弹出后是否仍有白条遮挡、命中错位、布局跳变
- `MotionDebugPanel` 是否能看到中文参数和实时诊断
- 陀螺仪授权、原生监听、样本时间是否合理

## 真人连线检查
- 浏览器是否验证
- Capacitor / 模拟器是否验证
- 登录、占卜师列表、牌组列表是否通
- 建会话、join-token、入房、揭牌同步、挂断是否通
- 无摄像头环境下是否能正确降级

## 回传要求
- 明确写出跑了哪些命令
- 明确写出在哪个环境验证
- 明确写出 iOS / Android 各自已验证项和未验证项
- 没跑的项直接写“未验证”
